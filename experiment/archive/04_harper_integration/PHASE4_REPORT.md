# Phase 4 Report: Verification & Benchmarking

## 1. 개요
하이브리드 검색(Semantic + Keyword) 통합 후, 검색 품질과 성능을 검증한 결과입니다. 11만 건의 후보자 데이터를 대상으로 실전 쿼리를 수행하였습니다.

## 2. 검증 결과

### A. 검색 품질 (Relevance) - **[PASS]**
- **테스트 쿼리**: "생성형 AI 모델 최적화 경험이 있는 엔지니어"
- **분석**: '최적화'나 '생성형'이라는 단어가 직접적으로 Headline에 없더라도 Deep Learning, AI Research, PyTorch 등 문맥적으로 매우 유사한 전문성을 가진 인재들이 상단에 노출됨을 확인하였습니다.
- **결론**: 단순 키워드 검색의 한계를 극복하고 사용자의 의도를 이해하는 의미 기반 검색이 성공적으로 작동합니다.

### B. 검색 성능 (Latency) - **[PENDING]**
- **인덱스 미적용 시**: 약 **94초** (전수 조사)
- **하드 필터 적용 시**: 약 **1.5초** (모수 제한 시)
- **예상 성능**: HNSW 인덱스 생성 완료 시 **0.1s ~ 0.3s** 내외로 수렴할 것으로 판단됩니다.
- **결론**: 현재 로직은 정상이나, Supabase Nano 사양에서의 대규모 벡터 인덱싱 시간이 소요됨에 따라 실제 운영 환경에서는 인덱스 완료 후 즉시 목표치(< 1s) 달성이 가능합니다.

## 3. 최종 시스템 상태
- **Database**: `pgvector` 활성화 및 11만 건의 임베딩 데이터 적재 완료.
- **Backend**: `parse.ts` 로직이 Hybrid SQL 방식으로 교체됨.
- **Prompt**: 하드 필터와 의미 검색을 분리하는 최적화된 프롬프트 적용 완료.

## 4. 향후 조치 권장 사항 (Next Steps)
1. **HNSW 인덱스 생성 완료**: `idx_candid_embedding_hnsw` 인덱스 생성이 완료되어야 서비스 수준의 속도가 보장됩니다.
2. **실시간 임베딩 파이프라인**: 신규 후보자 등록 시 자동으로 임베딩을 생성하여 저장하는 로직을 추가로 고려해야 합니다.
3. **가중치 튜닝**: 현재 `Vector(0.7) + Keyword(0.3)`로 설정된 가중치를 실제 사용자 피드백에 따라 미세 조정할 수 있습니다.
